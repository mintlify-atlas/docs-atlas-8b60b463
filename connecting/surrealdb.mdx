Beekeeper Studio supports SurrealDB connections over WebSocket (`ws`/`wss`) and HTTP (`http`/`https`). You can browse schemafull tables, filter and edit data, and run queries against your SurrealDB instance. SurrealDB support is in active development.

<Note>
  SurrealDB requires the **Ultimate** (paid) edition of Beekeeper Studio.
</Note>

<Warning>
  SurrealDB support is an early implementation. Core features work well, but some capabilities are still in progress. Check back frequently for updates.
</Warning>

## Connection details

| Field | Default | Description |
|---|---|---|
| Protocol | `ws` | Communication protocol: `ws`, `wss`, `http`, or `https` |
| Host | — | IP address or hostname of your SurrealDB instance |
| Port | `8000` | Port SurrealDB listens on |
| Authentication Method | — | See authentication options below |
| Username | — | SurrealDB username |
| Password | — | SurrealDB password |

## Connect from the app

<Steps>
  <Step title="Open the connection screen">
    Launch Beekeeper Studio and open the connection screen.
  </Step>
  <Step title="Select SurrealDB">
    Choose **SurrealDB** from the connection type dropdown.
  </Step>
  <Step title="Choose a protocol">
    Select your protocol from the dropdown: `ws` for unencrypted WebSocket, `wss` for encrypted WebSocket, `http`, or `https`.
  </Step>
  <Step title="Enter host and port">
    Fill in the **host** and **port** fields. The default port is `8000`.
  </Step>
  <Step title="Select an authentication method">
    Choose from the authentication methods provided by SurrealDB and enter your **username** and **password**.
  </Step>
  <Step title="Connect">
    Click **Connect**.
  </Step>
</Steps>
