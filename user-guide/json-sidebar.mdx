---
title: "JSON Sidebar: Explore Records and Relations"
description: "Use Beekeeper Studio's JSON sidebar to view any table row as formatted JSON, expand inline foreign key relations, and search fields with text or regex."
---

The JSON sidebar is a panel in Beekeeper Studio that displays the currently selected table row as formatted JSON. It is particularly useful for wide tables with many columns, complex nested schemas, and tables that store JSON data in text or JSON columns — giving you a readable, structured view without having to squint at a narrow grid cell.

## Opening the JSON sidebar

You can open the sidebar in two ways:

<CardGroup cols={2}>
  <Card title="Right-click a row" icon="mouse-pointer">
    Right-click any row in a table view or query result and select **See Details** to open the sidebar focused on that row.
  </Card>
  <Card title="Toolbar icon" icon="panel-right">
    Click the dock-right icon in the application title bar to toggle the JSON sidebar open or closed.
  </Card>
</CardGroup>

Once open, the sidebar updates automatically as you click different rows in the table.

## Browsing the JSON tree

The sidebar displays the selected row as a collapsible JSON tree. Each key-value pair is shown on its own line, and nested objects or arrays are expandable. Once you have found the data you want, copy the entire JSON or any subtree — it is standard JSON and pastes anywhere.

```json
{
  "id": 42,
  "name": "Acme Corp",
  "address": {
    "street": "123 Main St",
    "city": "Springfield"
  },
  "tags": ["enterprise", "active"]
}
```

## Viewing related records inline

If a column contains a foreign key, you can click it in the JSON sidebar to expand the related record inline. The referenced row's data appears nested beneath the foreign key value without opening a new tab or writing a join query.

This is especially useful in heavily normalized schemas — such as those generated by Ruby on Rails or Django — where a single record may reference several other tables.

<Info>
  Inline relation expansion works for foreign key columns where Beekeeper can detect the relationship from your database schema. If a relation is not expanding, check that a foreign key constraint is defined on the column.
</Info>

## Searching within the sidebar

The search box at the top of the JSON sidebar supports both plain text and regular expressions, so you can quickly locate a specific field or value within a large record.

<CardGroup cols={2}>
  <Card title="Fuzzy text search" icon="magnifying-glass">
    Type any text to filter the JSON tree to keys and values that match. Useful for finding a specific field name in a wide record.
  </Card>
  <Card title="Regular expressions" icon="code">
    Toggle regex mode to search using patterns, such as finding all fields whose values look like email addresses or ISO dates.
  </Card>
</CardGroup>

```text
-- Find fields matching an email pattern
[\w.+-]+@[\w-]+\.[a-z]{2,}

-- Find all keys that contain "id"
\".*id.*\":
```
