Beekeeper Studio is built and maintained by a small, independent team. Whether you're a paid subscriber looking for direct help or a community user seeking answers, there are several ways to get support, report bugs, request features, and connect with other users.

## Commercial support

If you have an active Ultimate edition subscription, you're entitled to direct email support from the Beekeeper Studio team.

<Card title="Email support" icon="envelope">
  Write to [support@beekeeperstudio.io](mailto:support@beekeeperstudio.io)

  The team responds during business hours (9–5 Mon–Fri) and sometimes outside those hours. Include your OS, app version, and a description of the issue. Attaching a screen recording or log file speeds up resolution significantly.
</Card>

<Note>
  Email support is available to subscribers of the **Ultimate edition**. If you're on the Community edition, the community resources below are your best option for getting help.
</Note>

## Submitting a bug report

When filing a bug report, include as much context as possible to help reproduce the problem quickly.

<Steps>
  <Step title="Check existing issues">
    Search [GitHub Issues](https://github.com/beekeeper-studio/beekeeper-studio/issues) to see if your bug has already been reported.
  </Step>
  <Step title="Collect debug information">
    - Check **Help → Show Developer Tools** for console errors.
    - Find your log files (see [data locations](/support/data-location) for paths by OS).
    - Record a screen capture of the issue if possible.
  </Step>
  <Step title="Open a new issue">
    Click [New Issue](https://github.com/beekeeper-studio/beekeeper-studio/issues/new/choose) and choose the **Bug Report** template. Fill in all the fields, attach your logs and recording, and submit.
  </Step>
</Steps>

<Tip>
  A screen recording is the single most helpful thing you can attach to a bug report for visual or hard-to-reproduce issues. See the [troubleshooting guide](/support/troubleshooting) for recommended recording tools on each OS.
</Tip>

## Feature requests

Have an idea for a new feature or improvement?

<Steps>
  <Step title="Search first">
    Check [GitHub Issues](https://github.com/beekeeper-studio/beekeeper-studio/issues) to see if someone has already requested the same feature. If so, add a thumbs-up reaction or leave a comment with your use case — this helps the team prioritize.
  </Step>
  <Step title="Open a feature request">
    Click [New Issue](https://github.com/beekeeper-studio/beekeeper-studio/issues/new/choose) and choose the **Feature Request** template. Describe the problem you're trying to solve and how you'd expect the feature to work.
  </Step>
</Steps>

## Contributing to Beekeeper Studio

Beekeeper Studio's Community edition is fully open source under the GPLv3 license and welcomes contributions of all kinds.

<AccordionGroup>
  <Accordion title="Contributing code">
    Fork the [GitHub repository](https://github.com/beekeeper-studio/beekeeper-studio), make your changes, and open a pull request. Include a description of what your change does — a GIF or screen recording is always welcome for visual changes. Read [CONTRIBUTING.md](https://github.com/beekeeper-studio/beekeeper-studio/blob/master/CONTRIBUTING.md) before you start.

    To run Beekeeper Studio locally:

    ```bash
    # Requires Node.js 20, npm, and Yarn
    git clone git@github.com:<your-username>/beekeeper-studio.git
    cd beekeeper-studio
    yarn install
    yarn run electron:serve
    ```
  </Accordion>
  <Accordion title="Contributing without code">
    You don't need to write code to contribute. You can help by:

    - Translating documentation (the docs site supports 10 languages)
    - Improving documentation for clarity or accuracy
    - Reproducing and confirming bug reports
    - Adding your use case or feedback to feature request issues
    - Answering questions in GitHub Discussions or Slack

    Read the [guide to contributing in 10 minutes without coding](https://github.com/beekeeper-studio/beekeeper-studio/issues/287) for more ideas.
  </Accordion>
  <Accordion title="Code of conduct">
    Beekeeper Studio is committed to maintaining an inclusive and welcoming community. All contributors and community members are expected to follow the [code of conduct](https://github.com/beekeeper-studio/beekeeper-studio/blob/master/code_of_conduct.md).
  </Accordion>
</AccordionGroup>
